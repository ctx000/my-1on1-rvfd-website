Step 1 of 4

Create your free
EmailJS account.

EmailJS is free for up to 200 emails per month — plenty for coaching applications. No credit card needed.

1

Go to EmailJS

Open a new tab and go to emailjs.com → click "Sign Up Free" in the top right corner.

2

Sign up with your Gmail

Use the same Gmail address you want coaching emails sent FROM. Click "Sign up with Google" — it's the fastest option.

3

Get your Public Key

Once logged in, click "Account" in the left sidebar → you'll see a box labeled "Public Key" — it looks like this:

Public KeyaBcDeFgHiJkLmNoPq

Copy it and paste it somewhere safe — you'll need it in Step 4.

📋 This key identifies your account. Keep it — but don't share it publicly.

Step 2 of 4

Connect your
Gmail account.

This is what EmailJS uses to actually send the emails. It takes about 3 clicks.

1

Go to Email Services

In the left sidebar click "Email Services" → then click the blue "Add New Service" button.

2

Select Gmail

You'll see a list of email providers. Click "Gmail" — it has the red Gmail icon.

3

Name it and connect

In the "Name" field type: RVFD Gmail

Then click "Connect Account" → select your Gmail → allow the permissions → click "Create Service".

✅ Once connected you'll see it listed under Email Services with a green dot. EmailJS will show you a Service ID — it looks like service_xxxxxxx. Copy and save this too.

Step 4 of 4 — Final Step

Paste your keys into
apply.html.

Open your apply.html file in any text editor (Notepad, TextEdit, VS Code — anything). Search for this block near the top of the <script> section:

  const EJS_PUBLIC_KEY      = 'YOUR_PUBLIC_KEY';
  const EJS_SERVICE_ID      = 'YOUR_SERVICE_ID';
  const EJS_TEMPLATE_IMMED  = 'YOUR_TEMPLATE_ID_1';
  const EJS_TEMPLATE_FOLLOW = 'YOUR_TEMPLATE_ID_2';
  const SCHEDULING_LINK     = 'YOUR_SCHEDULING_LINK';
  const AGREEMENT_LINK      = 'YOUR_AGREEMENT_LINK';
1

Replace each placeholder

Replace each value inside the quotes with what you collected in Steps 1–3:

YOUR_PUBLIC_KEY← from EmailJS Account page
YOUR_SERVICE_ID← from Email Services (service_xxxxx)
YOUR_TEMPLATE_ID_1← Template 1 ID (template_xxxxx)
YOUR_TEMPLATE_ID_2← Template 2 ID (template_xxxxx)
YOUR_SCHEDULING_LINK← your Calendly or booking URL
YOUR_AGREEMENT_LINK← your GitHub Pages URL + /Client_Coach_Agreement.docx
2

Example of what it looks like filled in

  const EJS_PUBLIC_KEY      = 'aBcDeFgHiJkLmNoPq';
  const EJS_SERVICE_ID      = 'service_abc12345';
  const EJS_TEMPLATE_IMMED  = 'template_xyz11111';
  const EJS_TEMPLATE_FOLLOW = 'template_xyz22222';
  const SCHEDULING_LINK     = 'https://calendly.com/csanchez/discovery';
  const AGREEMENT_LINK      = 'https://csanchez.github.io/rvfd/Client_Coach_Agreement.docx';
3

Save the file and upload to GitHub

Save your updated apply.html → go to your GitHub repo → click apply.html → click the pencil (edit) icon → paste the updated content → click "Commit changes".

That's it. Every time someone submits the application, EmailJS fires the confirmation email immediately. The 24-hour follow-up is scheduled automatically in the background.

4

Test it before going live

Submit a test application on your live site using your own email address. You should receive the confirmation email within about 30 seconds. Check your spam folder if it doesn't appear in your inbox.

⚠️ Note on the 24-hour follow-up: The follow-up fires from the client's browser 24 hours after they submit — so their browser tab needs to stay open (or be revisited) for it to send. For guaranteed delivery, EmailJS's paid plan ($9/month) supports server-side scheduling. The free version handles Email 1 perfectly and Email 2 in most cases.
